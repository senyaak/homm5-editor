# Object fields, measured

Generated by `node tools/object-schema.ts --md` from 152 shipped maps.
Do not edit by hand — re-run it instead.

Kinds are inferred from the values seen: `enum` is a short closed set and is
listed in full, `structure` has element children, `reference` carries an href.

**The enum lists are a lower bound, not the legal set**, and more maps will not
fix that. Adding the 15 original-campaign maps from the All_campaigns mod and
the maps inside the game's own paks took the monster sample from 3247 to 5802
and produced not one new enum value. So a field like `AttackType`, which is
`ATTACK_ANY` on all 5802, has whatever other values it has recorded nowhere in
any map — those would have to come from the game's own definitions. Treat these
lists as safe choices to offer, not as a rule to validate against.

**Field sets vary slightly between objects of one type.** `AdvMapMonster` shows
`DoesNotDependOnDifficulty` on 3242 of 3247 monsters in the sample set: the
older maps predate it. That is why a new object is built by cloning a real one
rather than from a fixed template — see `src/donors.ts`.
